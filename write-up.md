Design Rationale

This reflection tree is built to guide users through three progressive psychological shifts:

Agency (Axis 1)
Questions distinguish between internal vs external locus of control using real behavioral cues (reaction to success/failure).
Branching ensures:
High-agency users reinforce ownership
Low-agency users are gently nudged to notice choice

Contribution (Axis 2)
Focus shifts from control → value creation.
Questions reveal:

Contribution mindset (helping, initiative)
Entitlement mindset (recognition, fairness concerns)

Design choice: avoid moral judgment, instead reframe entitlement as natural but improvable.

Radius (Axis 3)
Expands perspective from self → others.
Uses progressive framing:

Self → Team → Individual → Customer

This mirrors self-transcendence theory, encouraging meaning through outward focus.

Key Design Decisions
1. Deterministic branching

All decisions use:

Explicit answer mapping
No ambiguity or scoring model
2. Fixed options only

Each option is:

Mutually exclusive
Psychologically meaningful
Realistic for tired users
3. Reflection tone
Non-judgmental
Observational, not prescriptive
“Wise colleague” voice
4. Progressive flow

Each axis builds on the previous:

Agency → enables Contribution
Contribution → enables Perspective expansion
Sources Used
Rotter — Locus of Control
Carol Dweck — Growth Mindset
Organ — Organizational Citizenship Behavior
Maslow (1969) — Self-Transcendence
Batson — Perspective Taking
Trade-offs
Chose clarity over complexity (no deep nested branches)
Limited nodes per axis to maintain usability
Avoided over-personalization to preserve determinism
Improvements (if more time)
Add deeper sub-branches for mixed signals
Introduce pattern-based summaries (multi-day)
Add adaptive difficulty (short vs long reflection paths)
Improve summary personalization using more state variables
